# Nova Dark

Theme sources. Build from the repo root with `docker compose run --rm build`;
see the root README for everything else.

```
nova-dark/
├── nova-dark-config.json   GENERATED from _palette.scss. Do not hand-edit;
│                           every build overwrites it.
├── NovaDark.qss            GENERATED by qtsass. Not tracked.
├── source/
│   ├── NovaDark.scss       entry point, imports the three layers below
│   ├── _palette.scss       every colour in the theme. The ONLY file allowed
│   │                       a hex literal; the build fails otherwise.
│   ├── _tokens.scss        semantic roles mapped onto palette primitives
│   └── _widgets.scss       the Qt stylesheet rules
├── icons/modern/           91 Phosphor SVGs, checked in and GENERATED
└── scripts/
    ├── generate-config.py       palette -> nova-dark-config.json
    └── download_phosphor_icons.py   palette -> the icon SVGs
```

## Changing a colour

Edit `source/_palette.scss`. Nothing else holds a colour.

- **Stylesheet colours** apply on the next build.
- **`nova-dark-config.json`** (qBittorrent's own palette, transfer-list states,
  log levels, pieces bar) is regenerated by every build.
- **Icon colours** live under the `$icon-` prefix. The SVGs are checked in, so
  they only change when you re-run the downloader:

  ```bash
  python src/nova-dark/scripts/download_phosphor_icons.py
  ```

  That needs an internet connection. A normal build does not.

Colours are not listed here on purpose. This file used to carry a status-colour
table that drifted until all nine values were wrong. `_palette.scss` is the
source of truth and it is commented.
